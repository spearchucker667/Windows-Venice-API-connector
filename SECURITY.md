# Security Policy

The full Venice Forge security model is maintained in [docs/SECURITY.md](docs/SECURITY.md).

## Reporting a Vulnerability

Do not include exploit details, API keys, tokens, or private user data in a public issue.

Use one of these routes:

- Open a GitHub issue labeled `security` and request a private maintainer discussion.
- If GitHub private vulnerability reporting is enabled for the repository, use that workflow.

The maintainer will triage reports for supported versions and coordinate disclosure before public details are posted.
